# Session Summary — Everything Built

## Auth System
- **Supabase Auth** with email+password (Google OAuth planned for later)
- Backend: `POST /auth/signup` (admin.createUser + auto-confirm + auto-login), `POST /auth/login`, `POST /auth/refresh`, `GET /auth/me`
- Signup creates `user_profiles` row + 4 `connected_apps` rows (gmail, gcal, gdocs, linear — all inactive)
- JWT verification via `supabase.auth.getUser()` (works with both HS256 and ES256)
- Auth middleware: `requireAuth` (blocks), `extractUserId` (optional)

## Swift Onboarding
- First launch → centered borderless window ("DANOTCH / WELCOME TO THE NOTCH" → signup/login form)
- `AuthManager` singleton: manages tokens, persists to `~/.danotch/auth.json`
- Auto-refreshes expired tokens before any API call (`ensureValidToken()`)
- Shows "Hi, {name}" greeting in the notch overview
- After auth → window closes, notch starts

## Chat + DB Persistence
- `POST /api/chat` with Anthropic Messages API + **tool-use loop** (max 5 iterations)
- User message saved to Supabase `messages` table (awaited)
- Assistant response saved fire-and-forget (`dbSave()`)
- Error/interrupted responses saved with `status: "failed"` + partial content + tools used
- Thread auto-created in `threads` table, title auto-generated by Claude after first response
- Title pushed to app via WebSocket for real-time update (shows "New Chat" initially)

## Thread History
- `GET /api/threads` — list user's past conversations
- `GET /api/threads/:id` — load messages for a thread
- AGENTS tab shows conversation history under "HISTORY" section with relative timestamps
- Clicking loads messages into chat view with `isFromHistory=true` (stays in history, doesn't appear in recents)
- Sending a follow-up flips `isFromHistory=false` (promotes to recents)

## Tools (3 categories)

### Local Tools (`src/tools/local.ts`)
| Tool | What | Details |
|------|------|---------|
| `bash_execute` | Run shell commands | 30s timeout, 1MB buffer, returns stdout+stderr |
| `web_search` | Search the web | DuckDuckGo HTML scraping, top 5 results, no API key |
| `web_fetch` | Fetch URL content | Strips HTML, handles JSON, 15s timeout, 5000 char limit |

### Scheduled Task Tools (`src/tools/scheduled.ts`)
| Tool | What |
|------|------|
| `create_scheduled_task` | Create cron/poll task with `notify_user` flag |
| `list_scheduled_tasks` | List all with schedule + status |
| `update_scheduled_task` | Toggle, edit schedule/prompt |
| `delete_scheduled_task` | Delete by ID |

### Tool Call UI
- Each tool call renders as a card in the chat: status icon → tool icon + label → input summary → output preview
- Spinner (dotted circle) while running, green checkmark when done
- Tool calls appear in sequence, final Claude response at the bottom

## Scheduled Tasks & Scheduler
- 30s tick loop (`setInterval`) queries `scheduled_tasks WHERE enabled AND next_run_at <= NOW()`
- Updates `next_run_at` immediately (prevents double-pickup), fires execution in background
- Re-fetches task before execution (catches delete/disable race condition)
- Two notification modes:
  - **Silent** (`notify_user=false`): saves result to `last_result`, no notification
  - **Conditional** (`notify_user=true`): wraps prompt with `[NOTIFY]/[SKIP]` instruction, Claude decides
- Scheduled tasks shown on HOME tab under "SCHEDULED" section (collapsible, expandable to show last output as markdown)
- Created via natural language in chat (Claude calls the tool)

## Notifications
- **Bell icon** in top bar with red dot unread badge
- **Notifications panel** (`.notifications` view state): grouped by `sourceId`
  - Group header: title, count badge, timestamp, expand chevron
  - Expanded: individual runs with timestamps, click to see full markdown output
  - Each run individually marked as read on expand
  - Pause/delete source task directly from notification group
- WebSocket push: `peek_notification` for conditional alerts, `notification` for regular
- **Peek animation**: soft notch grow (not full expand), shows ❗+ title, hover expands to show body + "VIEW ALL"
- Auto-dismiss: 4s if no hover, 2s after hover leaves

## Settings (persisted to `~/.danotch/settings.json`)
- Chat: openChatOnSend, restoreLastView, keepOpenInChat
- Display: calendarMode (off/mini/large), showMusic, musicSize, showBattery, showDotGrid, dotGridColor, dotGridOpacity
- Agents: showAgentLiveState, compactAgentRows

## Backend Infrastructure
- Request logging middleware (timestamp, method, path, auth status)
- Graceful shutdown: `stopScheduler()` + `notch.disconnect()` + 500ms force exit
- `DELETE /api/notifications/all` for bulk cleanup
